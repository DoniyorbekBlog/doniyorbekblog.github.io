Why This Topic Matters

Comprehensions are one of the most frequently used Python features in real projects. They allow you to write shorter, cleaner, and more expressive code. Mastering them is essential for moving from beginner-level code to professional-quality Python.

1. List Comprehension

Traditional Approach


numbers = [1, 2, 3, 4, 5]
squares = []

for n in numbers:
    squares.append(n * n)

print(squares)

Comprehension Version


numbers = [1, 2, 3, 4, 5]
squares = [n * n for n in numbers]

General Pattern:


[new_value for item in iterable]

2. Adding Conditions (Filtering)


numbers = [1, 2, 3, 4, 5, 6]
even_numbers = [n for n in numbers if n % 2 == 0]

The if condition filters elements before they are added to the new list.

3. Dictionary Comprehension

Dictionary comprehensions are used to transform or create dictionaries efficiently. 


names = ["Ali", "Sara", "John"]
name_lengths = {name: len(name) for name in names}

Pattern:


{key: value for item in iterable}

4. When NOT to Use Comprehensions

  • Logic is complex
  • Too many conditions
  • Hard to debug or read

Readability is always more important than writing fewer lines.

5. Comprehensions with if-else statement


    numbers = [1, 2, 3, 4, 5]
    all_s = [n*n if n%2==0 else n for n in numbers]
    print(all_s)
