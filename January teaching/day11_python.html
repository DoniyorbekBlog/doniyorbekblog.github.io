Problem 1: Square Sum Function

Complete the square sum function so that it squares each number in the list and returns the sum of the squared values. Example: for [1, 2, 2], the result is 9 because 1² + 2² + 2² = 9.

My Solution


def square_sum(numbers):
    result = 0
    for i in numbers:
        result += i*i
    return result

Best Solution


def square_sum(numbers):
    return sum(x ** 2 for x in numbers)

Detailed Explanation

My approach is fully correct: I initialize a variable result, iterate through each number, square it, and accumulate the sum. This is a classic, explicit way to solve the problem, and it clearly shows the logic step by step. It is especially readable for beginners.

The best solution is more compact and uses Python’s built-in sum() function together with a generator expression: (x ** 2 for x in numbers). This allows Python to compute each squared number lazily and then feed it directly into the summing process. As a result, the final code becomes shorter, cleaner, and more “Pythonic,” while still remaining very readable.

From a performance perspective, both solutions run with the same time complexity. However, the best solution avoids manual variable updates, reduces boilerplate, and follows modern Python idioms used in professional code.

Problem 2: Rearrange Digits in Descending Order

Create a function that takes any non-negative integer and returns a new integer with its digits arranged in descending order. This creates the highest possible value from those digits.

Examples:
Input: 42145 → Output: 54421
Input: 145263 → Output: 654321
Input: 123456789 → Output: 987654321

My Solution


def descending_order(num):
    num_list = list(str(num))
    num_list.sort(reverse=True)
    return int("".join(num_list))

Important Note


I cannot use:
    sorted_list = num_list.sort()

Because .sort() returns None.
Always sort the list first, THEN use it.

Detailed Explanation

My solution follows the correct sequence of steps: convert the number into a list of its digits, sort that list in descending order, and then join the digits back into a string before converting it to an integer. This method is clear, explicit, and easy to understand while still being efficient.

A key detail is the behavior of the .sort() method. In Python, .sort() modifies the list in place and returns None. This means I cannot write: 


num_list = list(str(num))
sorted_list = num_list.sort(reverse=True)  # WRONG

Such code results in sorted_list being None, which will cause errors later. I correctly handled this by calling .sort() directly on the list and then using that list immediately.

Python also provides a functional alternative using sorted(), which returns a new list instead of modifying the original. The more compact variant would be: 


def descending_order(num):
    return int("".join(sorted(str(num), reverse=True)))

This version is more concise and often considered more Pythonic. However, My version gives learners a clearer view of the underlying steps, so both are valid depending on the teaching goal.
